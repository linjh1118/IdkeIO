# 🚀 快速部署指南

## 5 分钟部署到 GitHub Pages

### 第一步：启用 GitHub Pages

1. 打开你的仓库：https://github.com/linjh1118/IdkeIO

2. 点击 `Settings`（设置）

3. 在左侧菜单找到 `Pages`

4. 在 `Build and deployment` 部分：
   - **Source（来源）**：选择 `GitHub Actions`
   - 点击保存

### 第二步：等待部署

1. 点击仓库顶部的 `Actions` 标签

2. 你会看到一个正在运行的工作流：`Deploy Jekyll site to Pages`

3. 等待 2-5 分钟，直到显示绿色勾号 ✅

### 第三步：访问网站

部署成功后，访问：

**https://linjh1118.github.io/IdkeIO/**

就这么简单！🎉

## 📍 关于 URL 路径

由于你已经有个人主页（`linjh1118.github.io`），这个项目会部署为**子路径**：

- 你的个人主页：`https://linjh1118.github.io/`
- 实验室主页：`https://linjh1118.github.io/IdkeIO/`

两个网站可以同时存在，互不干扰。

## 🔄 如何更新网站

以后修改网站内容后：

```bash
git add .
git commit -m "更新内容"
git push
```

GitHub Actions 会自动重新部署，2-5 分钟后更新就会生效。

## ❓ 遇到问题？

### 问题 1：Actions 显示失败

**解决办法：**
- 点击失败的工作流，查看详细错误日志
- 确保 `Settings` → `Pages` → `Source` 已选择 `GitHub Actions`

### 问题 2：网站显示 404

**解决办法：**
- 等待 5-10 分钟，GitHub Pages 首次部署可能需要一些时间
- 确认访问的 URL 是：`https://linjh1118.github.io/IdkeIO/`（注意末尾的斜杠）

### 问题 3：样式丢失或图片不显示

**解决办法：**
- 检查 `_config.yaml` 第 11 行，确保 `baseurl: "/IdkeIO"` 存在
- 强制刷新浏览器（Ctrl+F5 或 Cmd+Shift+R）
